# Alen's Hotwheels Garage – Website Guide

Welcome to your personal Hotwheels collection website!  
This guide explains how to update your inventory and keep your live site in sync.

---

## 📁 Folder Structure
Make sure your project folder looks like this:
your-folder/
├── index.html (main website)
├── cars_data.js (auto‑generated by the script)
├── List.xlsx (your Excel master file)
├── click this to convert excel to json.bat
├── generate_cars_data.py
├── Images/ (folder with all car photos)
└── README.md (this file)
---

## 🚀 How to Update Your Collection Locally

### Step 1 – Update the Excel file
1. Open `List.xlsx`.
2. **Add new cars** at the bottom (keep the same columns: `Series, Name, Condition, Card, Color, Price, Image - Filename, Availability`).
3. **For each new car**, place its photo in the `Images/` folder.  
   - Copy the exact filename (including extension, e.g. `bugatti.jpg`) and paste it into the `Image - Filename` column.
4. Save the Excel file.

### Step 2 – Run the batch file
Double‑click `click this to convert excel to json.bat`.  
This will:
- Read `List.xlsx`
- Generate a fresh `cars_data.js` file containing all your cars.
- Show a success message in the command window.

> **Note:** The script replaces the old `cars_data.js` – it does not append, but since your Excel contains the full list, the result is always complete.

### Step 3 – Preview the website
Open `index.html` in your browser.  
All your cars should appear, with images (or a “No image available” placeholder), search, and lightbox.

## 🌐 Deploy to the Web (Free – Netlify)

### Step 1 – Upload your project to GitHub
1. Create a free account on [GitHub](https://github.com).
2. Create a new repository (e.g., `hotwheels-garage`).
3. Upload your entire folder (including `index.html`, `cars_data.js`, `Images/`, etc.) to the repository.
   - **Important:** Do **not** upload `List.xlsx` or the Python/batch files if you want to keep them private. The website only needs `index.html`, `cars_data.js`, and the `Images/` folder.

### Step 2 – Connect Netlify to GitHub
1. Go to [Netlify](https://netlify.com) and sign up (use “Sign up with GitHub”).
2. Click **Add new site** → **Import an existing project**.
3. Choose your GitHub repository.
4. **Build settings**:  
   - Leave **Build command** empty.  
   - **Publish directory**: `.` (the root folder).  
   - In **Advanced settings**, you can add a redirect rule if needed.
5. Click **Deploy site**.

### Step 3 – Set the homepage
If Netlify asks for a **Landing page**, enter `index.html`.  
After a minute, Netlify will give you a live URL like `https://your-project-name.netlify.app`.  
Share this link with anyone – they can view your collection!

---

## 🔄 Updating the Live Site

Whenever you add new cars locally:
1. Repeat **Steps 1 & 2** (update Excel, run batch file, and copy new images into `Images/`).
2. **Push the changes to GitHub**:
   - Commit the updated `cars_data.js` and any new images in the `Images/` folder.
   - Push to your repository.
3. **Netlify automatically redeploys** – no extra steps needed. In a few seconds, your live site reflects the latest inventory.

## 🛠️ Troubleshooting

- **Images not showing?**  
  Make sure the filename in Excel exactly matches the filename in `Images/` (case‑sensitive on some servers).  
  Check the browser console (F12) for errors.

- **Website doesn’t update after pushing to GitHub?**  
  Netlify deployments may take a minute. Refresh your live site after the deploy finishes.

- **Python script error?**  
  Ensure `openpyxl` is installed: `pip install openpyxl`.  
  If Python isn’t in your PATH, edit the batch file to use the full path (e.g., `C:\Python312\python.exe generate_cars_data.py`).
